name: Keep files small and clean
description: User wants small, concise files — avoid large monolithic components
type: feedback
---

Always keep files small and concise. Break large Vue components into smaller ones. Avoid monolithic files with many lines.

**Why:** User values clean, readable code and maintainability. Large files are harder to review and maintain.

**How to apply:** Extract reusable sections into sub-components. Keep page-level components as thin orchestrators. Apply to all file types (Vue, PHP, etc.).
